我将为您在 `me.shetj.base.tools.debug` 包下设计并实现一套可视化的 Debug 调试工具。

### 🏗 架构设计

1.  **UI 模块 (`DebugActivity` & `DebugViewModel`)**：
    *   **设置页**：提供开关 HTTP 日志、修改 BaseUrl 等功能。
    *   **日志页**：展示日志列表，支持按关键字搜索、按级别过滤（Info/Error/Http/Behavior）。
    *   **独立性**：将 UI 日志和用户行为日志在展示层做区分，但底层存储可复用 `LogManager`。

2.  **核心功能扩展**：
    *   **HTTP 配置**：集成 BaseUrl 动态修改逻辑（需结合 Retrofit/HttpKit）。
    *   **日志读取**：在 `LogManager` 中增加读取日志文件的能力，供 UI 展示。

3.  **UI 交互**：
    *   使用 `ViewBinding` 或 `Compose` (鉴于 BaseKit 已引入 Compose，推荐使用 Compose 构建调试界面以减少样板代码)。
    *   提供悬浮窗入口或摇一摇呼出（可选，本次先实现 Activity 入口）。

### 📋 实施步骤

1.  **扩展 LogManager**：
    *   增加 `getLogFiles()`: 获取日志文件列表。
    *   增加 `readLogFile(file)`: 读取指定日志文件内容。
    *   优化 `LogRecord`: 确保日志格式易于解析。

2.  **实现 Debug 界面**：
    *   `DebugSettingsActivity.kt`: 调试设置主界面。
    *   `LogViewerActivity.kt`: 日志查看与搜索界面。
    *   如果项目支持 Compose，将优先使用 Compose 实现 `DebugScreen`。

3.  **HTTP 动态配置**：
    *   在 `DebugFunc` 或 `HttpKit` 中增加 `updateBaseUrl(url)` 方法。
    *   需要确保持久化存储（SP/DataStore）。

4.  **入口集成**：
    *   在 `DebugFunc` 中增加 `openDebugUI(context)` 方法。

### ⚠️ 确认事项
*   考虑到 BaseKit 已引入 Compose，我将使用 Jetpack Compose 来构建这套调试界面，这将更高效且现代。
*   BaseUrl 的修改通常需要重建 Retrofit 实例，我会在实现中处理这一逻辑。

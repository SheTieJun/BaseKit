# 局域网设备发现与通信工具 (LanKit)

## 概述

在同一个局域网中，自动发现对端设备、建立 TCP 连接并进行双向通信。

## 架构

```
网络层（network/lan/）
├── LanKit.kt                          # 统一入口（单例）
├── tcp/TcpLink.kt                     # TCP 通信层（服务端 + 客户端）
├── discovery/NsdHelper.kt             # NSD 设备发现与注册
├── model/LanDevice.kt                 # 设备信息模型
└── model/LanMessage.kt                # 通信消息模型
```

**流程**：
1. `LanKit.start(ctx, secretKey?)` → 启动 TCP 服务端 + 通过 NSD 注册本机服务（密钥哈希嵌入服务类型）
2. `LanKit.devices` (Flow) → 只发现同密钥的设备（已由 NSD 层过滤）
3. `LanKit.connect(device)` → 建立 TCP 连接，自动发送握手密钥
4. TCP 服务端二次校验握手密钥 → 双保险
5. `LanKit.send()` / `LanKit.broadcast()` → 发送消息
6. `LanKit.messages` (SharedFlow) → 接收消息

## 使用示例

```kotlin
// 1. 启动（带密钥校验：只有持有相同密钥的设备才能通信）
LanKit.start(applicationContext, secretKey = "my-app-key-2024")

// 或不带密钥（跳过握手校验，适合可信局域网）
LanKit.start(applicationContext)

// 2. 发现设备
lifecycleScope.launch {
    LanKit.devices.collect { devices ->
        // devices: List<LanDevice>
        updateDeviceList(devices)
    }
}

// 3. 连接到设备（自动发送密钥握手）
LanKit.connect(device)
    .onSuccess { /* 连接成功 */ }
    .onFailure { /* 处理错误，可能是密钥不匹配 */ }

// 4. 收发消息
LanKit.send(device, "Hello")
LanKit.broadcast("广播消息")

// 5. 监听消息
lifecycleScope.launch {
    LanKit.messages.collect { msg ->
        // msg.payload, msg.senderId, msg.type
        handleMessage(msg)
    }
}

// 6. 断开/停止
LanKit.disconnect(device)
LanKit.stop()
```

## 安全：双层校验

### 第一层 — NSD 发现过滤

`secretKey` 的 SHA-256 前 8 位作为服务类型后缀：`_basekit-{hash}._tcp.`

密钥不同 → 哈希不同 → NSD 服务类型不同 → 设备列表里直接看不到对方。

```
App A (key="project-a")        App B (key="project-b")
───────────────────────         ───────────────────────
NSD 注册: _basekit-a1b2c3d4    NSD 注册: _basekit-e5f6g7h8
     发现频道不同，互不可见
```

### 第二层 — TCP 握手校验

即使 NSD 被绕过（例如手动指定 IP 连接），TCP 握手层仍会拒绝密钥不匹配的连接。

```
设备 A (key="abc")              设备 B (key="abc")
─────────────────               ─────────────────
connect(B)
  └─ TCP 连接 ────────────────→ accept()
  └─ handshake("abc") ────────→ 校验: ✓
                                  └─ 开始收发
```

⚠️ 密钥哈希和握手消息均为明文传输，适用场景为**应用级区分**（防止不同 App 实例误连），不是防攻击的安全方案。需要端到端加密请在业务层使用 TLS。

## 技术细节

| 模块 | 技术方案 |
|------|---------|
| 设备发现 | Android NSD (Network Service Discovery) |
| 通信协议 | TCP + 长度前缀帧 (4 字节大端 + JSON) |
| 并发 | Kotlin Coroutines (Dispatchers.IO) |
| 序列化 | Gson |
| 端端口 | 自动分配 (ServerSocket(0)) |
| 设备标识 | Android ID |

## 注意事项

- `minSdk = 26`, NSD 需要 API 16+，完全兼容
- TCP 连接超时默认 5 秒
- 消息最大长度 1MB
- NSD 需要 Wi-Fi 处于同一局域网（不支持移动网络）

# 蓝牙端对端通信工具 (BtKit)

## 概述

通过经典蓝牙 RFCOMM（SPP 协议）实现设备发现、连接和双向通信。

## 架构

```
网络层（network/bt/）
├── BtKit.kt                           # 统一入口（单例）
├── transport/BtRfcommLink.kt          # RFCOMM 传输层（服务端 + 客户端）
├── discovery/BtDiscovery.kt           # 蓝牙设备发现
└── model/BtDevice.kt                  # 设备信息模型
```

**消息模型复用** [LanMessage](file:///Users/pc/Documents/Github/BaseKit/baseKit/src/main/java/me/shetj/base/network/lan/model/LanMessage.kt)，帧协议与 LanKit 一致（4 字节长度 + JSON）。

**流程**：
1. `BtKit.start(ctx)` → 启动 RFCOMM 服务端 + 开始设备扫描
2. `BtKit.devices` (Flow) → 持续输出附近蓝牙设备（含已配对）
3. `BtKit.connect(device)` → 建立 RFCOMM 连接
4. `BtKit.send()` / `BtKit.broadcast()` → 收发消息
5. `BtKit.messages` (SharedFlow) → 接收消息

## 使用示例

```kotlin
// 1. 启动
BtKit.start(applicationContext)

// 2. 发现设备
lifecycleScope.launch {
    BtKit.devices.collect { devices ->
        // devices: List<BtDevice>
        updateDeviceList(devices)
    }
}

// 3. 连接到设备
BtKit.connect(device)
    .onSuccess { /* 连接成功 */ }
    .onFailure { /* 处理错误 */ }

// 4. 收发消息
BtKit.send(device, "Hello")
BtKit.broadcast("广播")

// 5. 监听消息
lifecycleScope.launch {
    BtKit.messages.collect { msg ->
        handleMessage(msg)
    }
}

// 6. 断开/停止
BtKit.disconnect(device)
BtKit.stop()
```

## 权限要求

在 `AndroidManifest.xml` 中添加：

```xml
<!-- Android 12+ -->
<uses-permission android:name="android.permission.BLUETOOTH_SCAN" />
<uses-permission android:name="android.permission.BLUETOOTH_CONNECT" />

<!-- Android < 12 -->
<uses-permission android:name="android.permission.BLUETOOTH" />
<uses-permission android:name="android.permission.BLUETOOTH_ADMIN" />
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
```

运行时需自行申请权限（库不负责权限请求）。

## 技术细节

| 模块 | 技术方案 |
|------|---------|
| 通信协议 | Bluetooth Classic RFCOMM (SPP UUID) |
| 消息帧 | 与 LanKit 一致：4 字节大端长度 + JSON |
| 设备发现 | BroadcastReceiver + startDiscovery |
| 并发 | Kotlin Coroutines (Dispatchers.IO) |
| 序列化 | Gson |

## 与 LanKit 对比

| 特性 | LanKit (局域网) | BtKit (蓝牙) |
|------|----------------|--------------|
| 发现机制 | NSD 服务注册 | 经典蓝牙扫描 |
| 传输层 | TCP Socket | RFCOMM Socket |
| 密钥校验 | 双层（NSD 频道 + TCP 握手） | 未实现 |
| 消息模型 | LanMessage | 复用 LanMessage |
| 通信范围 | 同一 Wi-Fi | 10 米内 |

# 使用 Android Gradle 插件 (AGP) 升级助理

## 简介

Android Gradle 插件 (AGP) 升级助理是 Android Studio 中的一个内置工具，专门用于帮助开发者将其项目升级到更新的 AGP 版本。由于 AGP 经常发布新功能、新 API 以及对构建性能的优化，保持 AGP 更新对于利用最新功能和提高编译效率至关重要。升级助理不仅会自动修改构建文件，还会处理 AGP、Gradle 以及第三方插件之间的兼容性要求。

## 主要功能

1. **语法自动转换**：升级助理会自动将旧版 AGP 的构建文件语法转换为新版所需的格式，包括处理被废弃或不受支持的接口。
2. **兼容性管理**：
   - 自动检测并提示当前 AGP 版本所需的 Gradle 版本。
   - 检测并处理 AGP 与第三方 Gradle 插件之间的兼容性问题。
3. **分步指导与报告**：提供清晰的升级步骤，将升级过程分为“必需步骤”和“建议步骤”，并在升级后生成详细的报告，支持一键还原操作。

## 使用前提与项目设置

为了让升级助理能够最大程度地发挥作用，您的项目需要满足以下推荐设置：

1. **使用 Gradle 构建文件配置**：升级助理依赖于对构建文件的静态分析，因此请确保使用标准的 Gradle 构建文件（`build.gradle` 或 `build.gradle.kts`）。
2. **使用版本目录（Version Catalogs）**：推荐使用项目 `gradle` 子目录中的 `libs.versions.toml` 文件来集中管理依赖项和插件版本。
3. **声明式 DSL**：使用声明式领域专用语言（Groovy 或 Kotlin）。项目的配置越声明式，升级助理越容易找到需要调整的代码位置。
4. **代码备份**：在运行升级之前，**强烈建议**通过版本控制系统（如 Git）提交所有当前更改，或者备份项目。这确保了如果升级失败，您可以轻松回滚。

## 运行升级助理的步骤

1. **启动工具**：在 Android Studio 中，依次点击菜单栏的 **Tools > AGP Upgrade Assistant**，或者点击右下角弹出的升级通知提示。
2. **查看升级详情**：在弹出的工具窗口中，会显示当前 AGP 版本以及 Android Studio 支持的最新版本。
3. **选择升级步骤**：
   - 左侧面板会以树状图的形式展示升级涉及的各个步骤。
   - 这些步骤被分类为“必需更新”和“建议更新”。
   - 选中某个步骤后，主面板会显示详细的说明和更改原因。
4. **执行升级**：勾选需要的步骤后，点击 **Run selected steps**。Android Studio 会修改构建文件并尝试重新同步项目。
5. **验证升级**：同步成功后，查看升级报告。编译并运行项目的测试套件，确保升级没有破坏任何现有功能。
6. **提交更改**：确认一切正常后，将更改提交到版本控制系统。

## 错误排查指南

如果升级助理执行后项目同步失败，请按照以下步骤进行排查：

1. **检查错误信息**：首先查看构建输出中的错误日志。有时候错误很明显，可以直接在构建文件中手动修复。
2. **恢复原始状态**：如果错误难以定位，请使用版本控制（如 `git checkout`）或备份将项目恢复到升级前的状态。
3. **细分升级步骤**：
   - **逐步升级**：不要一次性执行所有建议的步骤。先只勾选“必需步骤”运行，如果成功，再逐个执行“建议步骤”，这样可以找出具体是哪一步导致了问题。
   - **增量升级**：如果跨越了多个大版本（例如从 7.0 升级到 8.2），建议先升级到中间版本。您可以在升级助理界面的下拉菜单中选择一个较低的目标版本，逐步进行升级。
4. **手动修复**：对于一些高度动态化或使用复杂脚本逻辑的构建文件，升级助理可能无法准确修改，此时需要您根据官方发行说明手动更新相应的 Gradle 脚本。

## 总结

AGP 升级助理是 Android 开发者维护项目构建系统的重要帮手。它通过自动化处理大部分繁琐的语法变更和兼容性对齐工作，极大地降低了版本升级的风险和成本。保持良好的项目结构（如使用 `libs.versions.toml` 和声明式构建脚本）并配合良好的版本控制习惯，是确保每次升级都能顺利完成的关键。

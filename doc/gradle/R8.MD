# R8 / ProGuard 使用说明（BaseKit）

本文档用于说明本项目中 R8（代码压缩/混淆/优化）相关的配置位置、常见问题与排查思路。

## 1. R8 是什么

- **R8**：Android 官方默认的代码压缩与混淆工具（AGP 默认启用），负责在 Release 构建时进行：
  - **Shrink**：移除未使用的类/方法/字段
  - **Optimize**：字节码优化（可选）
  - **Obfuscate**：符号混淆（类名/方法名变短）
  - **Resource Shrink**：配合 `shrinkResources` 移除未使用资源（由 AGP 执行，依赖 R8 输出）
- **ProGuard rules**：控制 R8 行为的规则文件（keep / dontwarn / keepattributes 等）

## 2. 本项目的 R8 开关位置

在 [app/build.gradle.kts](file:///Users/pc/Documents/Github/BaseKit/app/build.gradle.kts#L38-L49) 中：

- `release`：
  - `isMinifyEnabled = true`：开启 R8
  - `isShrinkResources = true`：开启资源压缩
  - `proguardFiles(...)`：指定规则文件
- `debug`：
  - 默认不开启 `minify`（当前配置仅指定了 `proguardFiles`，但未启用 `isMinifyEnabled`）

R8 规则文件位于：[proguard-rules.pro](file:///Users/pc/Documents/Github/BaseKit/app/proguard-rules.pro)

## 3. 常用规则（快速参考）

### 3.1 keep（保留类/成员，防止被裁剪/混淆）

常见场景：
- 反射（Gson/序列化框架/动态加载）
- JNI / WebView JSInterface
- 框架要求（某些 SDK 会在运行时按类名查找）

示例：

```pro
-keep class com.example.SomeApi { *; }
-keepclassmembers class * {
    @android.webkit.JavascriptInterface <methods>;
}
```

### 3.2 dontwarn（忽略缺失类/可选依赖）

适用于：
- 依赖库内部包含“可选功能”的引用（例如 Netty 支持多种压缩算法、不同平台 native 组件）
- 这些类在 Android 环境下不会被实际用到，但 R8 静态分析会扫描到引用并报错

示例（本项目曾遇到的 Netty / Ktor 场景）：

```pro
-dontwarn io.netty.internal.tcnative.**
-dontwarn com.github.luben.zstd.**
```

### 3.3 keepattributes（保留注解/泛型/行号等）

常用：

```pro
-keepattributes *Annotation*
-keepattributes Signature
-keepattributes SourceFile,LineNumberTable
```

## 4. 常见报错与处理方式

### 4.1 R8 Missing classes（缺失类导致 Release 构建失败）

典型提示：
> Missing classes detected while running R8 ... apply additional keep rules ... `missing_rules.txt`

处理步骤（推荐）：
1. 先定位 R8 输出的规则建议文件：
   - `app/build/intermediates/mapping/release/minifyReleaseWithR8/missing_rules.txt`
2. 将其中内容（通常是 `-dontwarn ...`）合并到你的规则文件：
   - [proguard-rules.pro](file:///Users/pc/Documents/Github/BaseKit/app/proguard-rules.pro)

注意点：
- `missing_rules.txt` 通常给的是 **dontwarn**，用于“忽略可选依赖缺失”。
- 若确实是“你自己业务必用的类缺失”，应优先补齐依赖，而不是 dontwarn。

### 4.2 MergeJavaResWorkAction（重复的 META-INF 文件）

典型提示：
> files found with path 'META-INF/xxx' ...

原因：
- 多个依赖 jar/aar 都包含同名的 `META-INF/*` 文件（索引/版本信息等），Android 打包合并失败。

处理方式：
- 在 [app/build.gradle.kts](file:///Users/pc/Documents/Github/BaseKit/app/build.gradle.kts#L64-L68) 的 `android { packaging { resources { ... } } }` 里排除冲突文件，例如：

```kotlin
packaging {
    resources {
        excludes += "META-INF/io.netty.versions.properties"
        excludes += "META-INF/INDEX.LIST"
    }
}
```

## 5. 如何验证 R8 配置是否正确

建议验证路径：
- Debug 快速验证：`./gradlew :app:assembleDebug`
- Release 严格验证（包含 R8）：`./gradlew :app:assembleRelease`

若 Release 能顺利产物输出且运行正常，说明 keep/dontwarn/packaging 配置基本可用。

## 6. 产物与排查路径（mapping / retrace）

R8 关键输出（Release）：
- mapping 文件（用于反混淆崩溃栈）：
  - `app/build/outputs/mapping/release/mapping.txt`
- 缺失类建议：
  - `app/build/intermediates/mapping/release/minifyReleaseWithR8/missing_rules.txt`

排查建议：
- 线上崩溃栈需要结合 `mapping.txt` 进行 retrace（Android Studio 或命令行均可）。

## 7. 建议实践

- **规则尽量最小化**：keep 过多会影响压缩收益；dontwarn 过多会掩盖真正缺失依赖问题。
- **优先补依赖，后用 dontwarn**：只有确认是“可选能力、不会在 Android 用到”时才 dontwarn。
- **每次引入重依赖/引擎类库后都跑一次 Release**：例如浏览器内核、网络引擎、加固、埋点 SDK 等。
